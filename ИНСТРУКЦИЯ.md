# Практика — Лекция 2 (C#)

Тренировка тем второй лекции: типы данных, условия, циклы, switch, pattern matching.  
Ориентир — **JetBrains Rider**.

---

## Быстрый старт в Rider

1. **File → Open** → выбери папку `csharp-lecture2-practice` (или файл `CsharpLecture2Practice.slnx`).
2. Дождись индексации и восстановления NuGet (строка статуса внизу).
3. В **Solution** (Alt+1) увидишь два проекта:
   - `CsharpLecture2Practice.Core` — **здесь ты пишешь решения**
   - `CsharpLecture2Practice.Tests` — **здесь автопроверка** (не меняй, только смотри)

---

## Где что лежит

| Файл / папка | Назначение |
|---|---|
| `Core/Lecture2Exercises.cs` | **Главный файл с задачами.** Ищи `// TODO: реализуй` |
| `Tests/Lecture2ExercisesTests.cs` | Тесты на каждую задачу. Показывают, что именно проверяется |
| `Core/Examples/` | **Готовые примеры** по каждой теме — подсматривай, не копируй слепо |
| `Core/Lecture2Exercises.Answers.cs` | Ответы на все задачи — **только если застрял** |
| `Core/appsettings.json` | Пример конфига (как на лекции: структура проекта) |

## Как запускать тесты в Rider

### Вариант 1 — один тест (удобнее всего)

1. Открой `Lecture2ExercisesTests.cs`.
2. Слева от `[Fact]` или `[Theory]` — **зелёный треугольник** в поле редактора (gutter).
3. Нажми на него → **Run** (или **Debug**, если хочешь отладку).

Пример: решил `TryParseUserChoice` → запусти тест `TryParseUserChoice_ValidatesInput`.

### Вариант 2 — все тесты сразу

- ПКМ по проекту `CsharpLecture2Practice.Tests` → **Run Unit Tests**
- Или ПКМ по классу `Lecture2ExercisesTests` → **Run Unit Tests**

### Вариант 3 — окно Unit Tests

- **View → Tool Windows → Unit Tests** (или **Alt+8**)
- Там дерево всех тестов, кнопки Run All / Rerun Failed

### Вариант 4 — терминал

```bash
dotnet test
```

Из корня solution. Удобно для CI или если Rider не подхватил проект.

## Solution, Project, Build — напоминание

- **Solution** (`*.sln` / `*.slnx`) — контейнер для нескольких проектов.
- **Project** (`*.csproj`) — отдельная сборка (Core = библиотека, Tests = тесты).
- **Build** — компиляция. Тесты не запустятся, если проект не собирается.
- **Reference** — Tests ссылается на Core (видно в `.csproj`: `ProjectReference`).

## Задачи — подробное описание

### Блок 1. Парсинг и валидация

**Файл:** `Lecture2Exercises.cs` → метод `TryParseUserChoice`  
**Пример:** `Examples/01_TryParse.cs`  
**Тест:** `TryParseUserChoice_ValidatesInput`

**Что сделать:**  
Разобрать строку в число. Успех только если:
- строка — целое число (`int.TryParse`);
- число от **0** до **3** включительно.

**Примеры:**
| Ввод | Результат |
|---|---|
| `"1"` | `true`, choice = 1 |
| `"abc"` | `false` |
| `"5"` | `false` |
| `"0"` | `true`, choice = 0 (выход в игре) |

**Связь с лекцией:** тот же код, что в Rock Paper Scissors после `Console.ReadLine()`.

---

### Блок 2. Switch и if-else

#### Задача 2a — `ChoiceToString`

**Пример:** `Examples/02_SwitchAndIfElse.cs`  
**Тест:** `ChoiceToString_ReturnsCorrectName`

**Что сделать:**  
По числу 1/2/3 вернуть `"Rock"` / `"Paper"` / `"Scissors"`.  
Любое другое число → `null`.  
Используй **switch expression** (`choice switch { 1 => "Rock", ... }`).

#### Задача 2b — `GetRoundResultText`

**Тест:** `GetRoundResultText_RockPaperScissors`

**Что сделать:**  
Определить исход раунда двух игроков (1..3):
- одинаковые → `"Draw"`
- пользователь победил → `"You win"`
- иначе → `"You lose"`

Правила победы (как на лекции):
- Rock (1) бьёт Scissors (3)
- Paper (2) бьёт Rock (1)
- Scissors (3) бьёт Paper (2)

### Блок 3. Цикл foreach

#### Задача 3a — `SumNumbers`

**Пример:** `Examples/03_Foreach.cs`  
**Тесты:** `SumNumbers_EmptyArray_ReturnsZero`, `SumNumbers_SumsAllElements`

**Что сделать:**  
Пройти по массиву через `foreach` и сложить все элементы.  
`[]` → `0`.

#### Задача 3b — `GetEvenNumbers`

**Тесты:** `GetEvenNumbers_FiltersCorrectly`, `GetEvenNumbers_AllOdd_ReturnsEmpty`

**Что сделать:**  
Вернуть **новый массив** только с чётными числами, порядок сохранить.  
Подсказка: `List<int>` + `foreach` + `Add`, в конце `.ToArray()`.

### Блок 4. Цикл for

#### Задача 4a — `CountMultiples`

**Пример:** `Examples/04_For.cs`  
**Тест:** `CountMultiples_Works`

**Что сделать:**  
Счётчик: сколько элементов массива делятся на `divisor` без остатка.  
Классический `for (var i = 0; i < numbers.Length; i++)`.

#### Задача 4b — `BuildSequence`

**Тест:** `BuildSequence_Works`

**Что сделать:**  
Создать массив длины `count`: `[0, 1, 2, ..., count-1]`.  
`count = 0` → пустой массив `[]`.

---

### Блок 5. Цикл while

#### Задача 5a — `Factorial`

**Пример:** `Examples/05_While.cs`  
**Тест:** `Factorial_Works`

**Что сделать:**  
Вычислить n! через `while`.  
`0! = 1`, `5! = 120`. Тип результата — `long` (чтобы не переполнился int).

#### Задача 5b — `CountHalvingSteps`

**Тест:** `CountHalvingSteps_Works`

**Что сделать:**  
Делить `number` на 2 (целочисленно), пока `number > threshold`.  
Вернуть количество делений.

Пример: `(20, 5)` → 20→10→5, делили **2 раза**, остановились (5 не больше 5).

### Блок 6. Value type vs Reference type

**Пример:** `Examples/06_ValueVsReference.cs`  
**Типы:** `Point` (struct) и `Player` (class) — внизу `Lecture2Exercises.cs`

#### Задача 6a — `DoublePointX`

**Тест:** `DoublePointX_ModifiesStruct`

**Что сделать:**  
Удвоить `point.X`. Параметр уже `ref Point` — меняй поле напрямую.  
**Важно:** struct копируется при передаче без `ref`, поэтому здесь `ref` обязателен.

#### Задача 6b — `AddBalance`

**Тест:** `AddBalance_ModifiesPlayerObject`

**Что сделать:**  
`player.Balance += amount`.  
Class передаётся по ссылке — изменения видны снаружи.

#### Задача 6c — `SafeLength`

**Тест:** `SafeLength_HandlesNullAndText`

**Что сделать:**  
Если `text == null` → `0`, иначе → `text.Length`.  
Строка — reference type, но **неизменяемая** (immutable).

---

### Блок 7. Pattern matching

#### Задача 7a — `Describe`

**Пример:** `Examples/07_PatternMatching.cs`  
**Тесты:** `Describe_Patterns`, `Describe_Null_ReturnsNullString`, `Describe_UnknownType_ReturnsUnknown`

**Что сделать:**  
Switch expression по типу значения:

| Значение | Результат |
|---|---|
| `null` | `"null"` |
| `int n` | `"number:{n}"` |
| `""` (пустая строка) | `"text:empty"` |
| `string s` | `"text:{s}"` |
| `bool b` | `"flag:{b}"` |
| anything else | `"unknown"` |

#### Задача 7b — `Grade`

**Тест:** `Grade_Works`

**Что сделать:**  
Шкала оценок через if-else или switch:
- 90–100 → `"A"`, 75–89 → `"B"`, 60–74 → `"C"`, 40–59 → `"D"`, 0–39 → `"F"`
- вне диапазона 0..100 → `"invalid"`

## Если застрял

1. Открой соответствующий файл в `Examples/`.
2. Посмотри тест — там видно вход и ожидаемый результат (`[InlineData(...)]`).
3. Поставь breakpoint (**Ctrl+F8**) в своём методе → **Debug** на тесте → смотри переменные.
4. В крайнем случае — `Lecture2Exercises.Answers.cs`.

---

## Частые ошибки

1. **Не тот файл** — код пишется в `Lecture2Exercises.cs`, не в `Examples/` и не в `Tests/`.
2. **Забыл `return`** — метод возвращает default (0, null, `""`).
3. **Struct без `ref`** — изменения не видны снаружи.
4. **Off-by-one в for** — `< numbers.Length`, не `<=`.
5. **CountHalvingSteps** — условие `number > threshold`, не `>=`.
